Общий слой процессов для рабочих процессов мультиагентной разработки, который позволяет агентам и людям видеть и контролировать долгоживущие процессы, такие как веб-серверы, сокращая копирование кода при работе с AI.

## Установка

### pip

```bash
pip install persistproc
```

## Доступные инструменты

| Инструмент | Описание |
|------|-------------|
| `ctrl` | Унифицированное управление процессами: запуск, остановка или перезапуск процессов |
| `list` | Список всех управляемых процессов и их статус. Может фильтровать по pid, команде или рабочей директории... |
| `output` | Получение захваченного вывода из процесса |

## Возможности

- Управление процессами во время выполнения без конфигурационных файлов
- Поддержка мультиагентных рабочих процессов
- Универсальный дизайн, не зависящий от инструментов
- Стриминг вывода процессов
- Получение логов ошибок
- Возможности перезапуска процессов
- Стандартный MCP сервер на http://127.0.0.1:8947

## Примечания

Сервер работает на http://127.0.0.1:8947. Запустите сервер командой 'persistproc serve' в отдельном терминале, затем запускайте процессы через 'persistproc <command>'. Наиболее полезен, когда вы хотите, чтобы агенты управляли процессами, а не только видели их вывод. Полная документация доступна на steveasleep.com/persistproc.
