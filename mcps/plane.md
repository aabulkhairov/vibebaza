MCP сервер, который позволяет языковым моделям взаимодействовать с Plane.so для управления проектами и задачами через API Plane, обеспечивая прямое взаимодействие с рабочими процессами управления проектами.

## Установка

### Smithery

```bash
npx -y @smithery/cli install @kelvin6365/plane-mcp-server --client claude
```

### Ручная установка

```bash
git clone https://github.com/kelvin6365/plane-mcp-server.git
cd plane-mcp-server
npm install
npm run build
```

## Конфигурация

### Claude Desktop

```json
{
  "mcpServers": {
    "plane": {
      "command": "node",
      "args": ["path/to/plane-mcp-server/build/index.js"],
      "env": {
        "PLANE_API_KEY": "your_plane_api_key_here",
        "PLANE_WORKSPACE_SLUG": "your_workspace_slug_here"
      }
    }
  }
}
```

## Доступные инструменты

| Инструмент | Описание |
|------|-------------|
| `list-projects` | Показывает все проекты в вашем рабочем пространстве Plane |
| `get-project` | Получает подробную информацию о конкретном проекте |
| `create-issue` | Создает новую задачу в указанном проекте с настраиваемыми свойствами |
| `list-issues` | Показывает и фильтрует задачи из проектов |
| `get-issue` | Получает подробную информацию о конкретной задаче |
| `update-issue` | Обновляет существующие задачи с новой информацией |

## Возможности

- Просмотр всех проектов в вашем рабочем пространстве Plane
- Получение подробной информации о конкретных проектах
- Создание новых задач с настраиваемыми свойствами
- Просмотр и фильтрация задач из проектов
- Получение подробной информации о конкретных задачах
- Обновление существующих задач с новой информацией

## Переменные окружения

### Обязательные
- `PLANE_API_KEY` - Ваш API ключ Plane.so для аутентификации
- `PLANE_WORKSPACE_SLUG` - Слаг вашего рабочего пространства Plane.so

## Примечания

Названия инструментов используют дефисы (например, list-projects), а не подчеркивания. Параметр assignees должен быть массивом строк с ID пользователей. Все операции, которые изменяют данные, требуют явного одобрения пользователя. Требует Node.js версии 22.x или выше.
