MCP сервер, который работает непосредственно на устройствах Android TV, обеспечивая управление через локальную сеть с помощью интеграции ADB для запуска приложений, получения списка программ и выполнения команд через естественный язык.

## Установка

### Сборка и установка

```bash
1. Build & install the app on your Android TV device or set-top box.
2. Ensure ADB debugging is enabled and working.
3. Launch the app — it will display the MCP Ready screen with device details and MCP server address.
```

## Доступные инструменты

| Инструмент | Описание |
|------------|----------|
| `launch_tv_app` | Запуск TV приложения по его имени пакета |
| `get_installed_tv_apps` | Получение списка установленных TV приложений |
| `take_screenshot` | Создание скриншота экрана устройства |
| `send_key_event` | Отправка события клавиши на устройство |
| `send_nav_key_event` | Отправка навигационного события клавиши (вверх, вниз, влево, вправо, выбор, назад, домой) |
| `type_text` | Ввод текста в текущее поле ввода |
| `tap_screen` | Нажатие на экран по определённым координатам |

## Возможности

- Прямая интеграция ADB без сопряжения
- MCP сервер работает на устройствах Android TV
- Работает с MCP Kotlin SDK 0.7.2 (официальный SDK)
- Управление устройствами в локальной сети через LLM клиентов
- Более простая настройка ADB по сравнению с телефонами - сопряжение не требуется
- Стабильное локальное соединение, которое не циклится
- Совместим с LLM клиентами, такими как Goose и Claude Desktop

## Примечания

⚠️ В разработке - Этот проект находится на ранней стадии разработки, и функции могут быть неполными или нестабильными. По соображениям безопасности это должно работать только в вашей локальной сети - подключение вашего TV к интернету представляет большой риск. Текущий метод связи использует SSE (устарел).
