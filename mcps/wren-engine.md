Wren Engine — это семантический движок для MCP клиентов и AI агентов, который предоставляет контекстно-зависимый доступ к корпоративным данным с пониманием бизнес-логики, управлением и поддержкой множества источников данных, включая BigQuery, PostgreSQL, MySQL, Snowflake и другие.

## Возможности

- Поддержка 11+ источников данных, включая BigQuery, PostgreSQL, MySQL, Snowflake, S3 и другие
- Семантическое понимание моделей данных с бизнес-контекстом
- Надёжные расчёты и агрегации для отчётности
- Определения бизнес-терминов (активный клиент, чистая выручка, отток клиентов)
- Разрешения и контроль доступа на основе пользователей
- Встраивается в любой MCP клиент или AI рабочий процесс
- Совместимость с современными стеками данных
- Семантико-ориентированный подход, позволяющий AI понимать модели данных
- Готов к управлению с ролями и контролем доступа
- Работает на Apache DataFusion для семантического SQL

## Примечания

В настоящее время в бета-версии с релизами каждые две недели. Проект состоит из 4 основных модулей: ibis-server (FastAPI веб-сервер), wren-core (семантическое ядро на Rust), wren-core-py (Python привязки) и mcp-server (реализация MCP сервера). См. mcp-server/README.md для инструкций по настройке MCP. Создан Canner и используется в Wren AI GenBI AI агенте.
