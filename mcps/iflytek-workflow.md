MCP сервер, который позволяет вызывать рабочие процессы iFlytek через MCP инструменты, обеспечивая интеллектуальное планирование рабочих процессов с поддержкой 14 типов узлов и различных режимов выполнения.

## Установка

### UVX из Git

```bash
uvx --from git+https://github.com/iflytek/ifly-workflow-mcp-server ifly_workflow_mcp_server
```

## Конфигурация

### Claude Desktop

```json
{
    "mcpServers": {
        "ifly-workflow-mcp-server": {
            "command": "uvx",
            "args": [
                "--from",
                "git+https://github.com/iflytek/ifly-workflow-mcp-server",
                "ifly_workflow_mcp_server"
            ],
            "env": {
                "CONFIG_PATH": "$CONFIG_PATH"
            }
        }
    }
}
```

## Возможности

- 14 типов узлов рабочего процесса, поддерживающих базовые, инструментальные, логические и трансформационные типы
- Последовательные, параллельные, циклические и вложенные режимы выполнения
- Сложный ввод/вывод переменных для гибкой передачи данных
- Механизм хуков для потоковой передачи и обработки в реальном времени
- Поддержка однократного и многократного взаимодействия с памятью контекста
- Поддержка нескольких моделей на основе архитектуры гибридного приложения Model of Models (MoM)
- Автоматическое выполнение рабочих процессов без необходимости ручного вмешательства

## Переменные окружения

### Обязательные
- `CONFIG_PATH` - Путь к файлу config.yaml, содержащему информацию о рабочих процессах

## Ресурсы

- [GitHub Repository](https://github.com/iflytek/ifly-workflow-mcp-server)

## Примечания

Требуется файл config.yaml с информацией аутентификации для рабочих процессов, включающий flow_id, api_key (в формате API Key:API Secret) и опциональные name/description. Пользователям необходимо создать и опубликовать рабочие процессы через платформу iFlytek по адресу https://xinghuo.xfyun.cn/botcenter/createbot и привязать их к приложению, чтобы получить необходимые учетные данные.
