Предоставляет шлюз для взаимодействия Generative AI систем с множественными Kubernetes кластерами через Model Context Protocol, обеспечивая комплексные операции с ресурсами Kubernetes и упрощенное управление мульти-кластерной инфраструктурой.

## Установка

### NPX

```bash
npx -y multicluster-mcp-server@latest
```

## Конфигурация

### Конфигурация MCP сервера

```json
{
  "mcpServers": {
    "multicluster-mcp-server": {
      "command": "npx",
      "args": [
        "-y",
        "multicluster-mcp-server@latest"
      ]
    }
  }
}
```

## Доступные инструменты

| Инструмент | Описание |
|------|-------------|
| `retrieve_hub_cluster_resources` | Получение ресурсов из хаб-кластера (текущий контекст) |
| `retrieve_managed_cluster_resources` | Получение ресурсов из управляемых кластеров |
| `connect_to_managed_cluster` | Подключение к управляемому кластеру с использованием указанной ClusterRole |
| `access_multi_cluster_resources` | Доступ к ресурсам в нескольких Kubernetes кластерах через Open Cluster Management |
| `retrieve_metrics_logs_alerts` | Получение и анализ метрик, логов и алертов из интегрированных кластеров |

## Возможности

- Получение ресурсов из хаб-кластера (текущий контекст)
- Получение ресурсов из управляемых кластеров
- Подключение к управляемому кластеру с использованием указанной ClusterRole
- Доступ к ресурсам в нескольких Kubernetes кластерах через Open Cluster Management
- Получение и анализ метрик, логов и алертов из интегрированных кластеров
- Комплексные операции с ресурсами Kubernetes
- Упрощенное управление мульти-кластерной инфраструктурой
- Интерактивное наблюдение за кластерами

## Переменные окружения

### Опциональные
- `KUBECONFIG` - Используется для доступа к конфигурации кластера, рассматривает настроенный кластер как хаб-кластер в мульти-кластерной конфигурации

## Примечания

Убедитесь, что kubectl установлен. По умолчанию инструмент использует переменную окружения KUBECONFIG для доступа к кластеру. В мульти-кластерной конфигурации он рассматривает настроенный кластер как хаб-кластер, получая доступ к остальным через него. Шаблоны промптов и MCP ресурсы для Open Cluster Management находятся в стадии планирования.
